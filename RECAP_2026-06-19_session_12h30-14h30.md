## 8. Synchronisation bidirectionnelle Commande client ↔ BL
**Commit `fde4ff5`** · Fichier : `CommercialDocs.jsx`

**Règle** : la dernière modification l'emporte sur l'autre pièce, toujours.

- Helper unique `syncLinesTo(targets, lines, patch, options)` qui recopie lignes + totaux + (optionnellement) infos client
- Verrou *frozenStatuses* (livré/payé/transformé) **n'est plus appliqué** sur le sync — l'utilisateur peut écraser une pièce figée
- Sauvegarde **commande client** → `syncCommandeDownstream` met à jour le BL
- Sauvegarde **BL** → `syncBLUpstream` met à jour la commande parente
- Champs synchronisés : article_id, ref, désignation, description, qté, unité, PU HT, TVA, totaux, ref constructeur, fournisseur, prix d'achat indicatif

## 10. BUG FIX — Clic sur les pastilles BC/BL n'ouvrait pas le doc
**Commit `29e8821`** · Fichier : `CommercialDocs.jsx`

**Cause** : `CommercialDocEditor` initialise son état interne `d` via `React.useState(doc)`. Le `useState` n'évalue son argument qu'au premier rendu — donc même quand le parent passe une nouvelle `doc` prop, l'éditeur conservait l'ancien `d`.

**Fix**
- `key={editing.id}` sur `<CommercialDocEditor>` → React démonte/remonte complètement quand l'id change
- `openDoc(id)` met aussi `setActiveType(full.type)` → la liste derrière le modal montre le bon onglet

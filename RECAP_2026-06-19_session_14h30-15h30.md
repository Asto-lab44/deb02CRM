## 5. Recherche d'article avancée — Catalogue + fallback IA
**Commit `baaca75`** · Fichier : `CommercialDocs.jsx` (`SmartArticleSearchModal`)

Nouveau bouton **🔍 Rechercher un article…** dans l'éditeur de devis/commande/BL, à côté de « + Ligne libre ».

**Workflow en 3 étapes** :

### Étape 1 — Recherche dans le catalogue
- Scoring tokens AND (ref + désignation + catégorie + description)
- Top 15 résultats triés par pertinence (matches complets en premier, puis prefix-match)
- Clic sur une suggestion → ajoute la ligne directement

### Étape 2 — Si zéro résultat → encart IA
- Encart jaune : *« Aucun article ne correspond à « X » dans le catalogue »*
- Bouton **🤖 Rechercher sur le web (IA) et créer l'article**
- Appelle `window.HubAi.suggestArticle(q)` ou `window.api.ai.searchArticle(q)` selon ce qui est branché

### Étape 3 — Brouillon éditable depuis la réponse IA
Pré-remplit un draft éditable :
- Désignation, Référence, Catégorie, Prix HT, Fournisseur, Description
- URL source affichée si fournie par l'IA
- Bouton **✓ Créer l'article et l'ajouter à la ligne** :
  - Crée l'article en BDD via `api.commercialArticles.create()`
  - Ajoute immédiatement la ligne au document
  - Recharge la liste catalogue locale

Si le module IA n'est pas branché : draft pré-rempli manuellement avec la requête comme désignation par défaut, l'utilisateur complète les champs.
